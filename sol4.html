让 SSS 成为有奇数个相邻单元格的单元格集合。不难发现， SSS 由边界单元格组成，不包括角落单元格。更准确地说

S={(i,j) ∣ (i∈{1,n}) ⨁ (j∈{1,m})=1}S = \{(i, j) \ \vert \ \left( i \in \{1, n\} \right) \ \bigoplus \ \left( j \in \{1, m\} \right) = 1 \}S={(i,j) ∣ (i∈{1,n}) ⨁ (j∈{1,m})=1}

观点 1. 我认为相邻边界单元格中不同颜色的单元格对的数量是偶数。事实上，如果我们沿着边界行走--这基本上形成了一个循环--从 (1,1)(1, 1)(1,1) 开始到 (1,1)(1, 1)(1,1) 结束，我们必须变换颜色的次数是偶数。

想法 2. 我们可以观察到，当翻转任何不在 SSS 中的单元格的颜色时， ∣A∣|A|∣A∣ 的奇偶性不会改变，因为这样的单元格有偶数个相邻的单元格。因此，如果我们固定边界单元格的颜色，设想将所有非边界单元格重新着色为白色， ∣A∣|A|∣A∣ 的奇偶性将保持不变。这意味着 ∣A∣|A|∣A∣ 的奇偶性完全取决于 SSS 中黑色（或白色）单元格数量的奇偶性。此外，我们还可以看到这些奇偶性实际上是相等的。

想法 3. 现在，我们只剩下计算使 ∣S∣|S|∣S∣ 包含偶数个黑色单元格的着色数。让我们考虑两种情况：

  1. SSS 中的所有单元格最初都是彩色的。在这种情况下，如果 SSS 中黑色单元格的数目是偶数，那么答案就是 2n⋅m−k2^{n \cdot m - k}2n⋅m−k ，否则就是 000 ，因为剩余单元格的颜色并不影响奇偶性。

    1. SSS 中至少有一个单元格没有着色。在这种情况下，答案为 2n⋅m−k−12^{n \cdot m - k - 1}2n⋅m−k−1 。这源于二项式系数的基本特征：

Cn0+Cn2+Cn4+⋯=Cn1+Cn3+Cn5+…C^0_n + C^2_n + C^4_n + \dots = C^1_n + C^3_n + C^5_n + \dotsCn0​+Cn2​+Cn4​+⋯=Cn1​+Cn3​+Cn5​+…

#include <iostream>
#include <vector>
 
using namespace std;
 
using ll = long long;
const ll mod = 1000000007;
 
ll binpow (ll a, ll n) {
    ll res = 1;
    while (n) {
        if (n & 1) {
            res *= a; res %= mod;
        }
        a *= a; a %= mod; n >>= 1;
    }
    return res;
}
 
void solve() {
    ll n, m, k; cin >> n >> m >> k;
    int good = 0, sum = 0;
    for (int i = 0; i < k; ++i) {
        int x, y, c; cin >> x >> y >> c;
        if ((x == 1 && y == 1) || (x == 1 && y == m) || (x == n && y == 1) || (x == n && y == m)) continue;
        if (x == 1 || y == 1 || x == n || y == m) {
            ++good; sum += c;
        }
    }
    if (good == 2 * (n + m - 4)) {
        if(sum%2)cout<<0<<end;
        else cout << binpow(2, n * m - k) << endl;
    } else {
        cout << binpow(2, n * m - k - 1) << endl;
    }
}
 
signed main() {
    int t; cin >> t;
    while (t--) solve();
}
